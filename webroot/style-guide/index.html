Go Lease It — Button System (Pencil Layout)

Layout and labels mirror Pencil. Runtime site button CSS stays unchanged; this style guide uses display-only parity helpers.

  • Contract in production: base `.btn` (default/large) and `.btn-sm` only.

Content Block by ID (Homepage CTA platter)

Homepage bottom CTA platter is rendered by Content Block ID in CTP. Example shortcode for RTE contexts:

{{block type="ContentBlock" id="123"}}

  • Layout binding keys: home_bottom_cta_block_id (fallback home_cta_block_id) and optional home_bottom_cta_wrapper_class.
  • Separate from footer CTA band: this content block does not use show_cta_platter; that toggle controls the global footer CTA band.

"Ready to Move Forward?" Footer CTA Band (Create + Enable)

This section is rendered in View/Elements/layout/footer.ctp and is primarily managed as a Content Block by ID.

Admin note: site-wide page settings are managed under Manage Default Page Fields.

Create / Update CTA Content (global)

  1. Create/update a Content Block containing full CTA HTML (recommended wrapper: <section class="cta-band cta-band--article">...</section>).
  2. Set Site.article_cta_content_block_id to that block ID for site-wide default.
  3. Optional page override: set cta_platter_content_block_id on a page to use a different CTA block there.

Enable / Disable Per Page

  1. In the target page, set field key show_cta_platter.
  2. Enabled values: 1, true, yes, on.
  3. Disabled values: 0, false, no, off.
  4. If key is missing or blank, default is enabled.

Fallback: if no CTA content block is configured/published, footer uses legacy Site.article_cta_* text/link settings.

RTE Control: Callouts

Admin editor includes a toolbar control labeled Callouts with action Insert Quote Callout to inject the required callout scaffold.

  1. Place cursor where callout should start.
  2. Open Callouts and choose Insert Quote Callout.
  3. Replace placeholder heading, quote, attribution, and body copy.
  4. Keep wrapper classes unchanged: article-split article-split--quote, article-callout article-callout--quote, article-quote.

Purpose: prevent fragile hand-authored structure in RTE while preserving the exact frontend contract.

Install + Bind Checklist (Active CMS Modules)

  1. Install the prototype instance in admin (Prototype Instances).
  2. Populate/publish ranked items and required fields.
  3. Bind the instance to page fields in the target layout.
  • Active starters used in this project: Home Process Steps, Home Stories, Home Testimonials.
  • Industries Served: use an industries instance (accepted slugs in layout fallback include industries-we-serve and industries_served), then set home_industries_instance_id or home_industries_instance_slug.
  • Home Process Steps (starter): install Home Process Steps (`home-process-steps`), then set home_process_instance_id or home_process_instance_slug.
  • Home Stories (starter): install Home Stories (`home-stories`), then set home_stories_instance_id or home_stories_instance_slug.
  • Home Testimonials (starter): install Home Testimonials (`home-testimonials`), then set home_testimonials_instance_id or home_testimonials_instance_slug.

Footer CTA Band (`cta-band`) Contract

Primitive: Content Block + Page field toggle (not a prototype).

Admin note: site-wide page settings are under Manage Default Page Fields.

  • Render path: View/Elements/layout/footer.ctp.
  • Enable key (per page): show_cta_platter (default is on when missing).
  • Primary content source: Site.article_cta_content_block_id (site default Content Block ID).
  • Optional page override: cta_platter_content_block_id.
  • Enabled values: 1, true, yes, on.
  • Disabled values: 0, false, no, off.
  • Legacy fallback: if no block is configured/published, uses Site.article_cta_heading, Site.article_cta_body, Site.article_cta_link, Site.article_cta_text.
  1. Create and publish CTA Content Block, then set Site.article_cta_content_block_id.
  2. Optional: set page-level cta_platter_content_block_id override.
  3. Set show_cta_platter on each page where you want explicit on/off control.
  4. Leave show_cta_platter blank/missing to use default-on behavior.
